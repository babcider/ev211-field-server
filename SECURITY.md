# 보안 정책

## 지원 범위

보안 수정은 최신 GitHub Release와 기본 브랜치에 제공됩니다. 운영자는 업데이트 전에 백업하고 최신 안정 릴리스로 갱신해야 합니다.

## 취약점 신고

공개 Issue에 비밀정보, 실제 서버 주소, 비밀번호, 인증서 개인키를 올리지 마세요. 저장소의 GitHub Security 탭에서 비공개 취약점 신고를 사용해 재현 절차와 영향을 전달해 주세요. 비공개 신고 기능이 보이지 않으면 저장소 관리자에게 공개되지 않는 연락 방법을 먼저 요청하세요.

## 보안 경계

- 이 서버는 신뢰할 수 있는 사설 LAN 전용입니다.
- 인터넷 공개와 공유기 포트 포워딩은 지원하지 않습니다.
- 같은 LAN의 사용자는 공개된 채널 목록을 보고 수신 토큰을 받을 수 있습니다.
- 송신과 무전기는 송신 비밀번호가 필요하고 관리 기능은 별도 관리자 비밀번호가 필요합니다.
- 인증 비밀번호는 HTTPS 8443으로만 전송합니다.
- LiveKit signaling은 내부망 TCP 7880을 사용하며 단기 JWT만 전달합니다. 음성 미디어는 WebRTC DTLS-SRTP로 암호화됩니다.
- Caddy 로컬 CA 개인키와 서버 데이터는 Docker volume에 있으므로 백업 파일도 비밀정보로 취급해야 합니다.
- 녹음 MP3에는 참가자의 음성이 포함되므로 녹음 고지·동의와 접근 권한, 보존 기간을 운영자가 관리해야 합니다.

## 공개 저장소에 올리면 안 되는 파일

- `.env`와 `/etc/ev211-field/ev211.env`.
- `initial-credentials.txt` 또는 비밀번호를 복사한 문서.
- Caddy `caddy_data`와 `caddy_config`.
- `field-root-ca.key`를 포함한 인증서 개인키.
- 운영 로그, 백업, 실제 내부망 구성도.
