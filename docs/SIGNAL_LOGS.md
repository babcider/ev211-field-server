# 송수신 신호 로그

EV211 Field Server는 방송과 무전기 사용 상태를 운영자가 사후 점검할 수 있도록 송수신 접속 이벤트를 SQLite에 30일간 보관합니다.

## 기록 범위

- 방송 수신·송신 및 무전기 토큰이 성공적으로 발급된 시각.
- 사용자가 LiveKit 방에 참가하거나 퇴장한 시각.
- 송신 마이크 트랙이 발행되거나 해제된 시각.
- 방송 또는 무전기 구분, 채널 번호, 서버 세대, 방과 트랙 이름, 접속 IP.

비밀번호, LiveKit JWT, 채널 비밀번호, 사용자 표시 이름, 음성 데이터와 RTP 패킷은 기록하지 않습니다. 내부 participant identity는 원문 대신 SHA-256 축약 해시만 저장합니다. 관리자 모니터와 서버 녹음용 기술 연결은 사용자 신호 로그에서 제외됩니다.

토큰 발급 기록은 접속 시도를, 참가·트랙 기록은 실제 연결 상태를 뜻합니다. 토큰을 발급받고 연결하지 않으면 토큰 기록만 남을 수 있으며, 수신 로그는 오디오 패킷 하나하나가 아니라 수신 참가자의 연결을 나타냅니다.

## 조회

서버 터미널에서 root 권한으로 최근 이벤트를 조회합니다. 기본값은 200건이고 최대 10,000건입니다.

```bash
sudo ev211ctl signal-logs
sudo ev211ctl signal-logs 1000
```

조회 결과는 최신순 탭 구분 표이며 시각은 UTC입니다. `direction`은 `receive`, `send`, `both` 중 하나이고 `scope`는 방송 `relay` 또는 무전기 `intercom`입니다.

## 보존과 삭제

field-api가 시작될 때 만료 로그를 즉시 정리하고, 실행 중에는 24시간마다 다시 정리합니다. 이벤트 발생 후 30일이 지난 행은 자동 삭제됩니다.

로그는 `/data/field.db`에 있어 `sudo ev211ctl backup` 백업에 포함됩니다. 오래된 백업에는 당시 신호 로그가 남을 수 있으므로 조직의 개인정보 보존 정책에 따라 백업 파일도 별도로 만료·삭제하세요. 접속 IP는 개인정보가 될 수 있으므로 서버와 백업의 root 접근 권한을 제한해야 합니다.
